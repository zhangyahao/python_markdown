### 什么是数据泄露

数据泄露是指在模型训练或调参过程中，无意中使用了本应只在最终评估时才接触的测试数据的信息，导致模型在评估时得到“虚假”的高分，但实际泛化能力差。

### 怎么解决

先将整个数据集划分为训练集和测试集（测试集严格不参与任何建模步骤）

```python
X_train, X_test, y_train, y_test = train_test_split(X, y, test_size=0.2)
```

### 为什么这样能防止泄露？

- 交叉验证内部：当使用`GridSearchCV` 会在训练集内部再次拆分（如5折），每折的验证集都来自训练集，从未接触过外部的测试集。使用确定模型时，使用
  `cross_val_predict`来交叉验证
   * 确定的模型
     ```python
        from sklearn.model_selection import cross_val_predict
        from sklearn.svm import SVC
        model = SVC(C=1, kernel='rbf')
        y_pred = cross_val_predict(model, X, y, cv=5)
     ```
   * GridSearchCV,使用cv参数来交叉验证，模型默认为5
     ````python
        from sklearn.model_selection import GridSearchCV
        grid = GridSearchCV(estimator=SVC(),param_grid= param_grid, cv=5, scoring='accuracy', n_jobs=-1)
     ````
- 参数选择：最佳参数是根据训练集内部的交叉验证得分选出的，测试集的信息在这个过程中完全没有被使用。
- 最终评估：只有调参完成后，才用测试集评估一次，这反映了模型对真正未见数据的泛化能力。

### 如果错误地将测试集也传入会怎样

* 测试集中的样本会被用来选择参数，相当于模型“提前知道”了测试集的答案。
* 最终测试集得分会虚高，无法反映真实性能。
* 这就是数据泄露，是建模过程中必须避免的错误
