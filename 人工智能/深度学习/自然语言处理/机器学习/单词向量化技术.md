### 详解

单词向量化技术是将自然语言中的单词转换为数学向量的过程，这些向量能够捕捉到单词的语义信息。向量化的目的
是使得机器能够理解和操作自然语言文本，因为计算机只能处理数值数据而不能直接理解文本。单词向量化的技术通
过将单词映射到高维空间，使得相似语义的单词在向量空间中相距较近。

#### 标签编码

1. 原理
   标签编码（Label Encoding） 是一种将分类变量（如“红”、“绿”、“蓝”）转换为整数标签（0, 1, 2）的方法。它给每个类别分配一个唯一的数字，但不考虑类别之间的顺序。
2. 优缺点
    - 优点：简单、省内存
    - 缺点：可能引入虚假顺序
3. 总结
    - 标签编码适合顺序类别，因为它保留了原有的顺序信息。
    - 标签编码不适合无序类别，因为它强加了虚假的顺序信息。
    - 适合树模型和有顺序的类别。

#### 独热编码

1. 原理：每个单词表示为一个稀疏的向量，其中向量的维度等于词汇表的大小。若某个位置对应的单词出现，则该位
   置为1，其他位置为0。
2. 优缺点：
    - 优点：简单，易于实现，无顺序假设
    - 缺点：膨胀维度，空间浪费，无法捕捉词语间的相似性，且不适合大规模数据集。
3. 独热编码 vs 标签编码

| 特性         | 标签编码 (Label Encoding)    | 独热编码 (One-Hot Encoding)                  |
|:-----------|:-------------------------|:-----------------------------------------|
| **转换结果**   | 一个整数列，如 `[2, 0, 1]`      | 多个二进制列，如 `[0,0,1]`, `[1,0,0]`, `[0,1,0]` |
| **类别数**    | 保留原类别数                   | 每增加一个类别就新增一列                             |
| **内存占用**   | 低（一列）                    | 高（多列）                                    |
| **是否隐含顺序** | 是（数字大小可能被模型误解为顺序关系）      | 否（各列独立，无顺序）                              |
| **适用模型**   | 树模型（决策树、随机森林、XGBoost）    | 线性模型、神经网络、距离模型（KNN、SVM）                  |
| **典型场景**   | 颜色、等级（低/中/高，但有序时也可用整数编码） | 城市、性别、产品类别（无序）                           |

#### 词袋模型

1. 原理：将文本中的单词转换为向量，向量的每一维表示词汇表中一个单词在文本中出现的频率。该方法忽略了单词的顺序
2. 优缺点：
    - 优点：简单，易于实现，适用于文本分类等任务。
    - 缺点：词汇表过大时会产生稀疏矩阵，无法捕捉单词之间的语法和语义关系，且没有对应的顺序关系
3. 不同版本：
   按照不同的度量标准，词袋模型可以认为有以下版本：
    - 二元特征：将文档中的每个单词转换为一个二元值（0 或 1），可以认为是简化版的词袋模型
    - TF：将文档中的每个单词转换为一个词频，正常的词袋模型版本
    - TF-IDF ：将词频和逆文档频率（IDF）结合起来的指标，可以认为是进阶版的词袋模型。一个词在文档中出现次数越多（TF高），并且在整个文档集合中越罕见（IDF高），它的
      TF-IDF 值就越高，说明这个词对这篇文档越重要。
4. IDF
   逆文档频率是信息检索与文本挖掘中常用的一种度量，用于评估一个词对整个文档集合的区分能力。
    1. 核心思想
        - 一个词如果在很多文档中都出现，那它很可能是一个“通用词”（如“的”、“了”、“是”），不太能帮助区分不同文档。
        - 相反，如果一个词只出现在少数文档中，那它就有很强的区分能力，更能代表该文档的主题。

#### n-gram

1. 原理：n-gram 是一种基于上下文的文本表示方法，它将文本分割成连续的 n 个词的组合（称为 n-gram）。
2. 优缺点：
    - 优点：n-gram 考虑到词语之间的相对位置和顺序，因此能够表示出更复杂的上下文信息。
    - 缺点：很容易产生维度灾难，n-gram 特征空间的维度随着 n 值的增大而急剧增加。
3. 对比

| 维度         | 词袋模型 (BoW)            | n-gram (n≥2)            |
|:-----------|:----------------------|:------------------------|
| **特征单元**   | 单个词                   | 连续 n 个词                 |
| **是否保留顺序** | 完全丢失                  | 保留局部顺序（短语级）             |
| **特征维度**   | 低（词汇表大小 V）            | 高（V + V² + ...，爆炸式增长）   |
| **稀疏性**    | 稀疏                    | 非常稀疏（大部分 bigram 出现次数为0） |
| **语义捕获能力** | 无法区分“A 喜欢 B”与“B 喜欢 A” | 可以捕获“not good”这类短语否定    |
| **计算复杂度**  | 低                     | 较高（特征多，内存大）             |
| **可解释性**   | 每个词独立贡献               | 短语特征更直观（如“机器学习”）        |
| **典型应用**   | 简单分类、主题建模             | 情感分析、机器翻译、拼写纠错          |
