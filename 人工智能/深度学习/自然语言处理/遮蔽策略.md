## 1. NLP 中的遮蔽策略概述

**问：什么是 NLP 中的遮蔽策略？**

- **定义**：在训练或推理时，对输入序列中某些位置的可见性进行人为控制。
- **常见类型**：
  - **MLM 随机遮蔽**（BERT）：随机选择 15% 的 token 进行遮蔽或替换，让模型预测原始词。
  - **因果遮蔽**（GPT）：上三角掩码，使当前位置不能看到未来 token。
  - **填充遮蔽**：忽略 padding 位置，避免注意力计算到无意义字符。
  - **跨模态/任务特定遮蔽**：如图像‑文本、实体遮蔽等。

------

## 2. 随机遮蔽（MLM）在训练中如何处理被遮蔽词？

**问：随机遮蔽时，对选中的 token 做了什么处理？**

- 从输入中随机选择 **15%** 的 token（不含特殊标记）。
- 对这 15% 的 token 按比例进行三种操作：
  - **80%**：替换为 `[MASK]`
  - **10%**：替换为一个随机 token
  - **10%**：保持不变（原词不动）
- **损失计算**：只在这 15% 的位置上计算交叉熵损失，标签为**原始 token**。

**目的**：

- 80% `[MASK]`：构造完形填空任务，学习双向语义。
- 10% 随机词：防止模型过度依赖 `[MASK]` 标记。
- 10% 原词：缩小预训练与微调之间的差异（微调时没有 `[MASK]`）。

------

## 3. `[MASK]` 的向量表示：为什么是可学习的？

**问：`[MASK]` 只是一个占位符，为什么它的嵌入向量是可学习的？**

- **核心原因**：`[MASK]` 的向量不仅用于表示“空缺”，更用于生成**注意力查询（Q）**。
  - 在自注意力中，`[MASK]` 位置的查询向量 `Q_mask` 决定它关注哪些上下文词。
  - 可学习的 `e_mask` 能让模型主动调整 `Q_mask`，从而更有效地收集有用信息。
- **固定向量（如全零）会导致**：`Q_mask` 为零，注意力分布均匀，削弱信息提取能力。
- **实验证据**：将 `[MASK]` 固定为零会使 MLM 准确率下降 1%~2%。

**类比**：`[MASK]` 不是方程中的未知数 `x`，而是求解 `x` 过程中的一个**可调初始猜测**。

------

## 4. 模型如何根据上下文预测 `[MASK]`？

**问：模型如何利用上下文信息预测被遮蔽的词？**

- 输入 `[MASK]` 的 ID 映射到可学习嵌入 `e_mask`。
- 经过多层 Transformer 自注意力，`[MASK]` 位置的输出向量 `h_mask` 融入了左右上下文。
- 一个线性分类层（MLM head）将 `h_mask` 映射为词表大小的概率分布。
- 模型选取概率最高的词作为预测。

**关键点**：

- 同一个 `e_mask` 在不同句子中，经过不同上下文调制，会得到完全不同的 `h_mask`。
- 因此，即使 `e_mask` 相同，模型也能预测出“学习”“理解”等截然不同的词，不会冲突。

------

## 5. 为什么要保留 10% 的原词（不替换）？

**问：那 10% 保持不变的原词，模型输入和标签都是原词，有什么意义？**

- **防止模型对 `[MASK]` 产生依赖**：如果模型只在见到 `[MASK]` 时才认真预测，那么微调时（没有 `[MASK]`）性能会下降。
- 强制模型在任何位置（无论是否是 `[MASK]`）都应具备**根据上下文推断语义**的能力。
- 尽管输入就是原词，但深层 Transformer 中信息已混合，模型不能简单复制；加上另外 90% 的扰动，最终学会的是真正的语义理解。

------

## 6. 推理阶段如何使用训练好的模型做完形填空？

**问：训练完成后，如何让模型对一句话中的 `[MASK]` 进行预测？**

1. 将待填空的句子中的词替换为 `[MASK]` 标记（例如 `"我爱[MASK]自然语言处理"`）。
2. 用分词器转为 token IDs（`[MASK]` 会被映射为固定的特殊 ID，如 BERT 中的 103）。
3. 输入模型，提取 `[MASK]` 位置输出的 logits。
4. 对 logits 做 softmax，取概率最高的 token ID。
5. 将该 ID 转回为文本（如“学习”）。

- **注意**：推理时不需要随机替换或保留原词的策略；可以同时预测多个 `[MASK]`。

------

## 7. `[MASK]` 的 ID 是由什么维护的？

**问：`[MASK]` 有一个 ID，这是模型内部维护的分词器吗？**

- 是的，**分词器（Tokenizer）** 维护一个固定的词表（vocab），其中包含所有普通 token 和特殊标记（`[CLS]`、`[SEP]`、`[MASK]`、`[PAD]` 等）。
- 每个特殊标记都有一个唯一的整数 ID，例如 BERT 中 `[MASK]` 的 ID 为 103。
- 模型内部的嵌入矩阵按行索引排列，`[MASK]` 的向量就是第 103 行（可训练）。
- 当我们在代码中写入 `tokenizer.mask_token_id` 时，返回的就是这个固定数字。

## 8. 模型如何处理中文分词与一词多义？

**问：中文中同一个字在不同语境下含义不同，模型如何维护分词和理解语义？**

- **分词**：现代 NLP 模型通常不依赖传统的“词”概念，而是使用 **子词分词（Subword Tokenization）**，如 BPE、SentencePiece。
  - 例如 `bert-base-chinese` 主要按**单字**切分（char‑level），每个汉字是一个 token。
  - 其他模型会合并高频相邻字为一个子词（如“自然”“语言”）。
- **一词多义解决**：通过 **Transformer 的自注意力机制** 产生**动态词向量**。
  - 同一个字（如“苹果”）在不同句子中，其输入嵌入向量虽然相同，但经过注意力层与上下文中其他词的交互后，得到的输出向量 `h_苹果` 完全不同。
  - 最终分类层会根据 `h_苹果` 输出合适的含义（水果或手机）。
- **未登录词（OOV）**：遇到未见过的词（如“打工人”），分词器会动态拆分为已知子词（“打工”“人”）。

------

## 总结

- **`[MASK]` 是可学习的**，目的是生成更好的注意力查询，并非简单的未知数。
- **随机遮蔽的三种操作**（80% `[MASK]`、10% 随机、10% 原词）平衡了预训练任务的有效性与泛化能力。
- **模型通过上下文调制同一个占位符**，可以在不同句子中预测不同词语，不会冲突。
- **推理时只需直接输入带 `[MASK]` 的文本**，取输出概率最高的词即可。
- **分词器维护特殊标记的固定 ID**，模型据此查找嵌入向量。
- **中文处理**采用子词分词 + Transformer 自注意力，从而灵活应对一词多义与未登录词。
