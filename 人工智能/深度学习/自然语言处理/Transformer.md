  ## 1. 背景与动机

  在 Transformer 出现之前，主流的序列建模方法是 RNN（LSTM、GRU）和 CNN。它们存在以下问题：

    - **RNN**：时序计算，难以并行；长距离依赖容易梯度消失/爆炸。
    - **CNN**：需要堆叠很多层才能捕获长距离依赖。

  Transformer 提出**完全基于注意力机制**的架构，抛弃循环与卷积，实现了：

    - **全局并行计算**
    - **直接建模任意距离的依赖**
    - **更强的可扩展性**（更大的模型、更多的数据）

  ------

  ## 2. 整体架构

  Transformer 采用 **编码器-解码器** 结构：

    - **编码器 (Encoder)**：将输入序列（如源语言句子）编码为连续的表示。
    - **解码器 (Decoder)**：基于编码器的输出，自回归地生成目标序列（如目标语言句子）。

  每个编码器/解码器由 **N 个相同的层** 堆叠而成（原论文 N=6）。

  **架构图示意**（编码器左，解码器右）：

  ```text
          输出概率
             ↑
          Softmax
             ↑
          线性层
             ↑
        解码器块 ×N
      ←───────────┐
      ↑           │
    编码器输出      │
      ↑           │
    编码器块 ×N    │
      ↑           │
    输入嵌入      输出嵌入（右移一位）
  ```

> 关键点：编码器和解码器之间通过**交叉注意力**连接。

  ------

## 3. 输入处理

- 输入 token 序列通过**嵌入层**转换为向量（维度 `d_model`）。
- 加上**位置编码**，因为注意力机制本身不包含顺序信息。

  ------

## 4. 自注意力机制 (Self-Attention)

### 4.1 核心思想

让每个 token 在编码时，能够“关注”到序列中所有其他 token，并根据相关性聚合信息。

### 4.2 QKV 三件套

对于输入矩阵 `X`（shape: `[seq_len, d_model]`），通过三个可训练矩阵 `W_Q`, `W_K`, `W_V` 计算：

  ```text
  Q = X * W_Q    # Query
  K = X * W_K    # Key
  V = X * W_V    # Value
  ```

### 4.3 注意力分数计算

text

  ```
  Attention(Q, K, V) = softmax( Q·K^T / √d_k ) · V
  ```

- `Q·K^T` 计算每对 token 之间的相似度。
- 除以 `√d_k` 缩放，防止点积过大。
- softmax 得到注意力权重（每行和为 1）。
- 加权求和 `V`，得到输出。

**示例**（两个 token）：

  ```text
  Q1=[0.2,0.8], K1=[0.5,0.1] → score11=0.18
  Q1·K2=0.62 → softmax → [0.39,0.61]
  output1 = 0.39*V1 + 0.61*V2
  ```

## 5. 多头注意力 (Multi-Head Attention)

单头注意力可能只关注一种特征模式，多头允许模型同时关注不同子空间的信息。

### 做法

- 将 `Q, K, V` 分别线性投影 `h` 次（h=8 是常用值），每次投影维度 `d_k = d_model / h`。
- 对每个投影分别计算注意力，得到 `h` 个输出。
- 拼接所有输出，再经过一次线性变换 `W_O`。

**公式**：

  ```text
  MultiHead(Q,K,V) = Concat(head1, ..., head_h) W_O
  head_i = Attention(Q·W_{Qi}, K·W_{Ki}, V·W_{Vi})
  ```

**好处**：不同头可以学习到不同的依赖关系（如局部语法、长距离语义）。

## 6. 前馈网络 (Feed-Forward Network)

每个注意力层之后紧跟一个全连接前馈网络（FFN），对每个位置独立且相同。

**结构**：两层线性变换 + ReLU 激活（现在常用 GELU）。

  ```text
  FFN(x) = max(0, x·W1 + b1)·W2 + b2
  ```

- 输入输出维度均为 `d_model`，中间隐藏层维度通常为 `4*d_model`。

**作用**：增加非线性表达能力，对每个位置的特征进行进一步变换。

  ------

## 7. 残差连接与层归一化

为了训练深层网络，每个子层（注意力层或 FFN）的输出都加上输入（残差连接），然后进行层归一化（LayerNorm）。

**子层输出**：

  ```text
  output = LayerNorm(x + Sublayer(x))
  ```

- **残差连接**：缓解梯度消失，让模型可以堆叠很多层。
- **层归一化**：稳定训练，加速收敛。

  ------

## 8. 位置编码 (Positional Encoding)

由于自注意力是置换等变的（不关心顺序），需要显式注入位置信息。

### 原始 Transformer 使用正弦/余弦函数：

  ```text
  PE(pos, 2i)   = sin(pos / 10000^(2i/d_model))
  PE(pos, 2i+1) = cos(pos / 10000^(2i/d_model))
  ```

- `pos` 是位置索引，`i` 是维度索引。
- 优点：可以外推到比训练时更长的序列。

### 其他位置编码

- 可学习位置编码（BERT、GPT 使用）
- 相对位置编码（Transformer-XL、RoPE）

  ------

## 9. 掩码机制 (Masking)

在注意力计算时，通过设置某些位置的分数为 `-inf`，使得 softmax 后权重为 0。

### 9.1 填充掩码 (Padding Mask)

- 忽略 padding 位置，避免模型关注无意义的 `<PAD>` token。

### 9.2 因果掩码 (Causal Mask / Look-ahead Mask)

- 用于解码器中的**自注意力**，确保当前位置只能看到它之前的 token（不能看到未来）。
- 形状：上三角矩阵，对角线及以下为 0，以上为 `-inf`。

### 9.3 交叉注意力中的掩码

- 编码器输出没有掩码。
- 解码器查询编码器时，通常只应用填充掩码。

  ------

## 10. 编码器 (Encoder)

编码器由 N 个相同层堆叠，每层包含两个子层：

1. **多头自注意力**（无掩码，允许看到整个输入序列）。
2. **前馈网络**。

每个子层后接残差连接 + 层归一化。

**输入 → 编码器 → 输出上下文向量**（作为解码器的 K 和 V）。

  ------

## 11. 解码器 (Decoder)

解码器也由 N 个相同层堆叠，每层包含三个子层：

1. **掩码多头自注意力**：使用因果掩码，防止看到未来的输出 token。
2. **交叉注意力 (Cross-Attention)**：Q 来自解码器上一层的输出，K、V 来自编码器的输出。
    - 允许解码器关注输入序列的不同部分。
3. **前馈网络**。

每个子层后也都有残差连接 + 层归一化。

**自回归生成**：解码器在生成第 `t` 个 token 时，输入是前 `t-1` 个真实 token（训练时）或已生成的 token（推理时）。

  ------

## 12. 训练与推理

### 12.1 训练

- **Teacher Forcing**：解码器的输入是真实的目标序列（右移一位，并在开头加入 `<SOS>`）。
- **损失函数**：交叉熵损失，只计算目标序列中非 padding 位置的 token。
- **优化器**：Adam，配合学习率预热（warmup）与衰减。
- **正则化**：Dropout、Label Smoothing。

### 12.2 推理（自回归解码）

- 初始输入：`<SOS>`。
- 循环：将当前已生成的序列输入解码器，预测下一个 token。
- 选择策略：贪心、束搜索（Beam Search）。
- 遇到 `<EOS>` 或达到最大长度时停止。

  ------

## 13. 总结

| 特性       | Transformer 实现方式 |
|:---------|:-----------------|
| 全局依赖捕获   | 自注意力机制，任意两位置直接交互 |
| 并行计算     | 无循环结构，序列长度维度全并行  |
| 位置信息     | 位置编码（正弦/可学习）     |
| 深层网络训练   | 残差连接 + 层归一化      |
| 表达能力增强   | 多头注意力 + 前馈网络     |
| 解码时的因果约束 | 因果掩码             |

Transformer 的成功开启了 **“注意力即一切”** 的时代，衍生出 BERT、GPT、T5 等一系列大模型，成为现代 NLP 乃至多模态任务的基石。
