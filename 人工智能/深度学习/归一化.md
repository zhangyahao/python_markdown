## 1. 作用

归一化（Normalization）是深度神经网络中至关重要的技术，用于稳定训练、加速收敛、提升泛化能力。  
在神经网络里，每一层输出的数据，数值大小、分布范围都不一样。如果不处理，网络训练起来就像“有的学生用100分制，有的用5分制”，
乱套了。归一化就是强行把这些数据拉到同一个标准尺度上，让训练更稳定、收敛更快

### 1.1 符号约定

- 输入特征符号 $ x∈R ^{N*C*H*W}  $
    - N：批量大小（batch size）
    - C：通道数（channels）
    - H,W：空间维度（高、宽）
- 归一化的值： $\hat{x}$
- 输出：$ y=γ \hat{x}+β$，其中 γ,β 为可学习的仿射参数

## 2. Batch Normalization (BN) 批归一化

### 2.1 原理

BN 在每个通道上，**跨样本计算均值和方差**。即对同一通道的所有样本、所有空间位置进行聚合。  
把不同样本的同一通道数据进行计算。
### 2.2 特点

* 依赖 batch size：batch 越大，统计量估计越准；batch=1 时失效。
* 引入样本间依赖：具有轻微的正则化效果。
* 加速训练：允许使用更高的学习率。

### 2.3 适用场景

* 大批量图像分类（ResNet、Inception）
* 目标检测（通常配合 SyncBN 跨卡同步

### 2.4 代码演示

```python
import paddle.nn as nn

bn = nn.BatchNorm2D(num_features=128)
output = bn(x)  # x shape: [N, 128, H, W]
``` 

## 3. Layer Normalization (LN) 层归一化

### 3.1 原理

LN 在每个样本上，**跨所有通道**和**空间位置**计算均值和方差。即对单个样本的所有特征进行归一化。  
分类后的确定特征的通道值进行计算。与整体无关
### 3.2 特点

* 完全不依赖 batch size：单样本即可，适用于 RNN、Transformer。
* 混合所有通道信息：可能弱化通道间的独立统计特性。
* 无正则化效果（不引入样本间噪声）。

### 3.3 适用场景

* 自然语言处理（Transformer、BERT）
* 循环神经网络（RNN、LSTM）
* 在线学习、小 batch 场景

### 3.4 代码示例

```python
import paddle.nn as nn

ln = nn.LayerNorm(normalized_shape=[128, 32, 32])  # 对 [C,H,W] 整体归一化
output = ln(x)
```

## 4. Instance Normalization (IN) 实例归一化

### 4.1 原理

IN 在每个样本的每个通道上独立计算均值和方差。即归一化范围是单样本单通道内的所有空间位置。  
实例”：指一个样本（一张图片、一个句子等）。在深度学习中，一个 batch 里有 N 个样本，每个样本就是一个“实例”。  
“归一化”：把数据分布拉回均值为 0、方差为 1 的标准范围。  
所以实例归一化 = 对每个样本单独做归一化，而且更进一步：每个通道也单独算。  
### 4.2 特点

* 完全解耦样本间和通道间依赖。
* 保留每个通道的独立统计特性，丢弃全局对比信息。
* 对小 batch 友好，batch size = 1 时仍有效

### 4.3 适用场景

* 图像风格迁移（AdaIN、CycleGAN）
* 纹理识别、生成对抗网络（GAN）的生成器
* 少样本学习、当 BN 不适用的小批量任务

### 4.4 代码示例（PaddlePaddle）

```python
import paddle.nn as nn

in_layer = nn.InstanceNorm2D(num_features=128)
output = in_layer(x)
```

## 5. Group Normalization (GN) 组归一化

### 5.1 原理

GN 将通道维度分成若干组（每组C//G 个通道），在每个样本的每组内计算均值和方差。它是 LN（G=1）和 IN（G=C）的广义形式。    
将数据类分组，计算组内均值和方差
### 5.2 特点

* 与 batch size 解耦，适用于小批量训练。
* 超参数 G：平衡 LN 和 IN 的行为。常用 G=32 或每组 16~32 通道。
* 计算开销与 BN 相当。

### 5.3 适用场景

* 目标检测、语义分割（显存受限，batch size 小）
* 视频理解、3D 卷积网络
* 任何 BN 因 batch 过小而失效的场景

### 5.4 代码示例（PaddlePaddle）

```python
import paddle.nn as nn

gn = nn.GroupNorm(num_groups=32, num_channels=128)
output = gn(x)
```

## 6. 对比总结

### 6.1 如何选择？

* 大批量训练（batch ≥ 32）且任务为判别式 → BN（效果最好，有正则化）
* 序列模型（NLP）、RNN、Transformer → LN
* 风格迁移、纹理生成 → IN
* 小批量视觉任务（目标检测、分割，batch = 1~8） → GN（或 IN，但 GN 通常更优）

### 特殊情况

* Batch size = 1：BN 不可用，选择 GN、IN 或 LN。
* 需要跨卡同步：使用 SyncBN（多卡通信）可近似大批量 BN 效果
