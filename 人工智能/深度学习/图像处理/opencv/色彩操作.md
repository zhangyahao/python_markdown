1. 彩色图片转换其他色彩`cvtColor`
   `cv2.cvtColor(src, code[, dstCn])`
   参数说明:
    - `src`: 输入图像，通常是一个 NumPy 数组
    - `code`: 转换代码，指定从哪种颜色空间转换到哪种颜色空间
    - `dstCn`: 可选参数，目标图像的通道数，默认为0，表示通道数会自动从 src 和 code 推断
      注意：
    - opencv读取到的图片是**bgr**格式
    - OpenCV 提供了超过150种颜色空间转换方法，不一定所有的颜色转换都支持可以使用
      `flags = [i for i in dir(cv2) if i.startswith('COLOR_')]`查看所有支持的颜色转换类型
    - `cv2.cvtColor()` 是一个高度优化的函数，通常比自己手动遍历像素要快得多。它内部利用了 SIMD
      指令集等硬件加速技术，因此在处理大量图像时，应尽量使用此函数而非自行实现
    - `数据类型`：确保输入图像的数据类型是 uint8 或 float32 等 OpenCV 支持的类型，否则可能导致转换失败。
    - `减少拷贝`：cvtColor 会返回一个新的图像数组。如果内存紧张或需要高性能，可以考虑使用 `cv2.UMat`
      或就地转换（如果可能），但通常直接使用返回值是最简单的做法。

3. 直方图均衡化
   直方图均衡化是一种自动改善图像对比度的图像处理技术。简单来说，它能将图像的像素亮度调整得更均匀，从而让图像暗部的细节显现出来，亮部的细节也不
   至于过曝，整体看起来更清晰。  
   它的核心算法是累积分布函数（CDF）。OpenCV的cv2.equalizeHist()函数正是基于此，通过以下4个步骤高效地完成直方图均衡化：
     1. 计算输入图像的灰度直方图。
     2. 将直方图归一化，使所有频数之和为255。
     3. 对归一化后的直方图进行累加，得到累积分布函数（CDF），并将其作为像素映射的查找表。
     4. 遍历图像每个像素，通过查找表将原灰度值映射到新的值，生成新图像。
   
   ### 全局直方图均衡化 `cv2.equalizeHist(src)`    
      示例：   
      ```python
        import cv2
        
        # 读取灰度图像
        img = cv2.imread('input.jpg', 0)
        
        # 对灰度图像进行直方图均衡化
        equ = cv2.equalizeHist(img)
        
        # 保存或显示结果
        cv2.imwrite('output.jpg', equ)
   
      ```
      **注意**：对彩色图像的处理：cv2.equalizeHist() 只适用于灰度图。如果需要对彩色图像进行处理，通常的做法是将其转换到 YUV 或 HSV
      色彩空间，然后只对其亮度（Y或V）通道进行均衡化，最后再合并通道并转换回 BGR 色彩空间。

   ### 自适应直方图均衡化 `cv2.createCLAHE()`
      cv2.equalizeHist()应用广泛，但它是一种全局方法，在某些情况下效果有限。为了解决这个问题，OpenCV提供了更先进的自适应直方图均衡化（AHE）
      及其改进版 CLAHE（限制对比度自适应直方图均衡化）。  
      区别：
 
      | 特性         | 全局直方图均衡化                           | 自适应直方图均衡化 (AHE)                 | 限制对比度自适应直方图均衡化 (CLAHE)              |
      |:-----------|:-----------------------------------|:--------------------------------|:------------------------------------|
      | **核心函数/类** | `cv2.equalizeHist()`               | `cv2.createCLAHE()` (是CLAHE的实现) | `cv2.createCLAHE()`                 |
      | **处理方式**   | **全局**：基于整个图像的直方图信息。               | **局部**：将图像划分为多个小块，分别进行均衡化。      | **局部 + 限制**：在AHE的基础上，对每个小块的对比度进行限制。 |
      | **主要优势**   | 原理简单，计算速度快。                        | 能有效提升图像的**局部对比度**，增强更多细节。       | 在增强局部对比度的同时，**有效抑制噪声**的放大，效果更自然。    |
      | **主要缺点**   | 在光照不均或图像局部差异大时，效果不佳，可能导致部分区域过曝或欠曝。 | 容易**放大图像噪声**，特别是在灰度变化平缓的区域。     | 计算量相对更大，需要调整参数以获得最佳效果。              |
      
      *  参数调优  
         `cv2.createCLAHE(clipLimit,tileGridSize)`  主要有两个参数可以调整：
           - clipLimit：对比度限制的阈值。值越大，图像的对比度增强效果越强，但同时也可能放大噪声；值越小，效果越温和。默认值是 40.0
           - tileGridSize：分块网格的大小，将图像划分为 (tileGridSize[0], tileGridSize[1]) 的小块。例如，(8, 8) 表示将图像分成8x8的小块。块越小，对局部细节的适应能力越强，但计算量也越大。默认值是 (8, 8)。
   
      * 示例：
        ```python
            import cv2

            # 1. 以灰度模式读取图像
            img = cv2.imread('low_contrast_image.jpg', 0)
            
            # 2. 应用全局直方图均衡化 (GHE)
            equ = cv2.equalizeHist(img)
            
            # 3. 应用 CLAHE
            # 创建CLAHE对象，可以设置参数，这里使用默认值
            clahe = cv2.createCLAHE(clipLimit=2.0, tileGridSize=(8,8))
            # 应用CLAHE
            clahe_img = clahe.apply(img)
            
            # 4. 显示并保存结果
            cv2.imshow('Original Image', img)
            cv2.imshow('Global Equalization (GHE)', equ)
            cv2.imshow('CLAHE Result', clahe_img)
            
            cv2.waitKey(0)
            cv2.destroyAllWindows()
            
            # 如果需要保存图片
            # cv2.imwrite('global_equalized.jpg', equ)
            # cv2.imwrite('clahe_equalized.jpg', clahe_img)
        ```
      * 总结
        `cv2.equalizeHist()` 是快速、便捷的全局增强工具，适合光照整体欠佳或过亮的图像。而 `cv2.createCLAHE()` 则是更强大、精细的局部自
        适应方案，能更好地处理复杂光照和细节丰富的图像，有效抑制噪声，是图像预处理和增强任务中更优的选择。
