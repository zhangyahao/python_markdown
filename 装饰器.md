## Python 函数装饰器详解

**装饰器（Decorator）** 是 Python 中一种强大而优雅的语法特性，它允许你在**不修改原有函数代码**的情况下，动态地**增加、扩展或修改
**函数的功能。装饰器本质上是一个**可调用对象**（通常是函数或类），它接受一个函数作为参数，并返回一个新的函数（或修改后的原函数）。

### 1. 为什么需要装饰器？

- **代码复用**：将与核心业务无关的通用功能（如日志、计时、权限校验）抽离出来，应用到多个函数上。
- **遵循开闭原则**：对扩展开放，对修改封闭。避免直接修改已有函数的代码，降低引入 bug 的风险。
- **语法简洁**：使用 `@decorator` 语法糖，让代码更直观、易读。

### 2. 装饰器的基本实现

装饰器通常是一个嵌套函数结构，外层函数接收被装饰函数，内层函数包装额外功能并调用原函数。

python

```
def my_decorator(func):
    def wrapper(*args, **kwargs):
        # 在原函数执行前添加额外操作
        print("调用函数前执行...")
        result = func(*args, **kwargs)   # 调用原函数
        # 在原函数执行后添加额外操作
        print("调用函数后执行...")
        return result
    return wrapper
```

使用 `@` 语法糖将装饰器应用到目标函数上：

python

```
@my_decorator
def say_hello(name):
    print(f"Hello, {name}!")
```

等价于：

python

```
say_hello = my_decorator(say_hello)
```

调用效果：

python

```
say_hello("Alice")
# 输出：
# 调用函数前执行...
# Hello, Alice!
# 调用函数后执行...
```

### 3. 常用示例

#### 示例1：计时器装饰器

python

```
import time

def timer(func):
    def wrapper(*args, **kwargs):
        start = time.time()
        result = func(*args, **kwargs)
        end = time.time()
        print(f"{func.__name__} 执行耗时: {end - start:.4f}秒")
        return result
    return wrapper

@timer
def slow_function():
    time.sleep(1)
    print("任务完成")

slow_function()
```

#### 示例2：日志装饰器

python

```
def logger(func):
    def wrapper(*args, **kwargs):
        print(f"调用函数 {func.__name__}，参数: args={args}, kwargs={kwargs}")
        return func(*args, **kwargs)
    return wrapper

@logger
def add(a, b):
    return a + b

print(add(3, 5))
```

### 4. 带参数的装饰器

如果装饰器本身需要参数（例如指定日志级别），则需要再嵌套一层函数。

python

```
def repeat(times):
    def decorator(func):
        def wrapper(*args, **kwargs):
            for _ in range(times):
                result = func(*args, **kwargs)
            return result
        return wrapper
    return decorator

@repeat(times=3)
def greet(name):
    print(f"Hello {name}")

greet("Bob")   # 会打印三次 Hello Bob
```

等价于：

python

```
greet = repeat(times=3)(greet)
```

### 5. 保留原函数元数据

使用装饰器后，原函数的元信息（如 `__name__`、`__doc__`）会被 `wrapper` 覆盖。可以使用 `functools.wraps` 装饰器来保留这些属性。

python

```
from functools import wraps

def my_decorator(func):
    @wraps(func)          # 关键：将 func 的元数据复制到 wrapper
    def wrapper(*args, **kwargs):
        """这是 wrapper 的文档"""
        print("before")
        return func(*args, **kwargs)
    return wrapper

@my_decorator
def example():
    """这是 example 的文档"""
    print("hello")

print(example.__name__)   # 输出 'example'，而不是 'wrapper'
print(example.__doc__)    # 输出 '这是 example 的文档'
```

### 6. 类装饰器

装饰器也可以是一个类，通过实现 `__call__` 方法使实例可调用。

python

```
class CountCalls:
    def __init__(self, func):
        self.func = func
        self.count = 0

    def __call__(self, *args, **kwargs):
        self.count += 1
        print(f"调用次数: {self.count}")
        return self.func(*args, **kwargs)

@CountCalls
def say_hi():
    print("Hi!")

say_hi()  # 调用次数: 1
say_hi()  # 调用次数: 2
```

### 7. Python 内置的常用装饰器

- **`@staticmethod`**：将类中的方法定义为静态方法。
- **`@classmethod`**：将方法定义为类方法，第一个参数为类本身。
- **`@property`**：将方法转换为属性访问，可定义 getter/setter。
- **`@functools.lru_cache`**：为函数提供缓存功能，优化重复计算。

### 8. 多个装饰器的叠加

装饰器可以叠加使用，执行顺序从下往上（离函数定义近的先执行）。

python

```
@decorator_a
@decorator_b
def func():
    pass

# 等价于 func = decorator_a(decorator_b(func))
```

### 总结

装饰器是 Python 中实现**切面编程**（AOP）的利器，它让代码更加模块化、可维护。理解装饰器的核心是**函数也是对象**
，可以像普通变量一样传递和返回。
