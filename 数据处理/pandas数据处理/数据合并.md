两个方法一个concat一个merge  
### concat
`concat`可以合并两个或多个DataFrame
- 函数
```python
pd.concat(
objs, # 合并数据，列表
axis=0, # 合并方向。默认是纵向合并
join='outer', # 索引按交集（inner）或并集（outer）合并
ignore_index=False, # 是否忽略索引合并
copy=True, # 是否复制
)
```
`objs`:接收多个Series，DataFrame的组合。表示参与连接的pandas对象的列表的组合。无默认。
`join`:接收inner或outer。表示其他轴向上的索引是按交集（inner）还是并集（outer）进行合并。默认为outer。
###  merge 
merge 数据库的 join一样的merge函数可以实现表连接  
- 函数
```python

pd.merge(
     left, # 左表
     right, # 右表
     how='inner',# 连接方式
     left_on=None, # 左表的连接字段
     right_on=None, # 右表的连接字段
     left_index=False, # 左表的连接索引
     right_index=False, # 右表的连接索引
     on=None, # 连接字段
     sort=False #表示是否根据连接键对合并后的数据进行排序。默认为False。
```
`how`接收inner，outer，left，right。表示数据的连接方式。默认为inner
`on`接收string或sequence。表示两个数据合并的主键（必须一致）。默认为None。
`left_index`接收boolean。表示是否将left参数接收数据的index作为连接主键。默认为False。
`right_index`接收boolean。表示是否将right参数接收数据的index作为连接主键。默认为False。
注意：当两个数据的关联字段字段名不同时，可以使用left_on和right_on去指定。on就不用写了
