# Pandas 索引方法 `at`、`loc`、`iloc` 、`iat`区别详解

Pandas 提供了多种数据选择（索引）方法，每种方法在用途、性能和使用场景上有所区别。正确理解它们能避免常见错误，提高代码效率。

## 1. 方法概览

| 方法     | 基于      | 返回类型               | 适用场景            | 是否支持赋值 |
|:-------|:--------|:-------------------|:----------------|:-------|
| `[]`   | 列名/切片   | Series 或 DataFrame | 快速选择列或行切片       | 是      |
| `loc`  | 标签      | Series 或 DataFrame | 按标签选择多行/多列      | 是      |
| `iloc` | 整数位置    | Series 或 DataFrame | 按位置选择多行/多列      | 是      |
| `at`   | 标签      | 标量（单个值）            | 快速访问单个单元格       | 是      |
| `iat`  | 整数位置    | 标量（单个值）            | 快速访问单个单元格（位置索引） | 是      |
| `ix`   | 混合（已弃用） | Series 或 DataFrame | 不推荐使用，避免歧义      | 是      |

## 2. 详细说明与示例

### 2.1 `loc` – 基于标签的索引

- **语法**：`df.loc[row_label, col_label]`
- **特点**：
    - 行和列都使用**标签**（索引名/列名）。
    - 支持切片、列表、布尔数组等。
    - 切片包含两端（闭区间）。
    - 当选择多个元素时返回 DataFrame，选择单个标量时返回 Series 或标量。
- **示例**：

python

```
import pandas as pd
df = pd.DataFrame({'A': [1,2,3], 'B': [4,5,6]}, index=['a','b','c'])

# 选择单个元素
df.loc['b', 'A']        # 输出 2

# 选择行切片和列列表
df.loc['a':'b', ['A','B']]   # 返回 DataFrame，包含行 a,b 和列 A,B

# 布尔索引
df.loc[df['A'] > 1, :]       # 返回 A 列大于 1 的所有行
```

### 2.2 `iloc` – 基于整数位置的索引

- **语法**：`df.iloc[row_position, col_position]`
- **特点**：
    - 行和列都使用**从 0 开始的整数位置**（类似 Python 列表索引）。
    - 支持切片、列表、整数等。
    - 切片不包含结束位置（左闭右开，类似 Python 切片）。
- **示例**：

python

```
# 选择单个元素
df.iloc[1, 0]        # 第二行第一列，输出 2

# 选择行切片和列列表
df.iloc[0:2, [0,1]]  # 返回前两行和所有列

# 选择最后一行
df.iloc[-1, :]
```

### 2.3 `at` – 基于标签的快速标量访问

- **语法**：`df.at[row_label, col_label]`
- **特点**：
    - **专门用于访问单个单元格**，比 `loc` 更快。
    - 行和列必须使用**单个标签**（不能是列表、切片）。
    - 直接返回标量值（或 Python 对象，如列表）。
    - 赋值时直接替换整个单元格内容，不会试图展开可迭代对象。
- **示例**：

python

```
# 取值
val = df.at['b', 'A']      # 2

# 赋值（存储列表）
df.at['b', 'A'] = [10, 20]  # 单元格现在存储一个列表
```

### 2.4 `iat` – 基于整数位置的快速标量访问

- **语法**：`df.iat[row_position, col_position]`
- **特点**：
    - 与 `at` 类似，但使用整数位置。
    - 比 `iloc` 快，专门用于单个标量。
- **示例**：

python

```
df.iat[1, 0] = 99          # 修改第二行第一列为 99
```

### 2.5 `[]` – 方括号索引

- **语法**：`df['col']` 或 `df[2:5]`
- **特点**：
    - 选择单个列返回 Series，选择多个列返回 DataFrame（使用列表）。
    - 行切片基于标签（如果索引是整数可能混淆），但通常不推荐用于行选择。
    - 赋值时可能引发链式索引警告。
- **示例**：

python

```
df['A']                    # 选择列 A
df[['A', 'B']]             # 选择多列
df['a':'b']                # 基于标签的行切片（包括 a 和 b）
```

## 3. 核心区别对比

| 特性          | `loc`               | `iloc`              | `at`          | `iat`         |
|:------------|:--------------------|:--------------------|:--------------|:--------------|
| **索引依据**    | 标签                  | 整数位置                | 标签            | 整数位置          |
| **允许的键**    | 标签、切片、列表、布尔         | 整数、切片、列表、布尔         | 单个标签          | 单个整数          |
| **返回类型**    | Series/DataFrame/标量 | Series/DataFrame/标量 | 标量（Python 对象） | 标量（Python 对象） |
| **切片行为**    | 包含结束标签              | 不包含结束位置             | 不支持           | 不支持           |
| **性能（单个值）** | 一般                  | 一般                  | 快             | 快             |
| **赋值列表/对象** | 可能引发长度错误            | 可能引发长度错误            | 直接存储对象，安全     | 直接存储对象，安全     |

## 4. 常见陷阱与注意事项

### 4.1 `loc` 赋值可迭代对象时可能引发错误

如：

python

```
df.loc[0, 'col'] = ['a', 'b']   # 可能报错：ValueError: Must have equal len keys and value
```

因为 `loc` 试图将列表的两个元素分配给某两列（即使只选了一列），导致长度不匹配。
 **解决方案**：使用 `at` 进行单个单元格赋值：

python

```
df.at[0, 'col'] = ['a', 'b']   # 正确，列表整体存入单元格
```

### 4.2 链式索引警告

避免使用 `df[df.a>0]['b'] = 5` 这样的链式索引，因为可能操作的是视图而非副本，赋值可能失效。应改用 `loc`：

python

```
df.loc[df.a > 0, 'b'] = 5      # 正确
```

### 4.3 视图 vs 副本

- `loc`/`iloc` 返回的结果可能是视图或副本，取决于操作。赋值时应确保操作的是原始 DataFrame。
- `at`/`iat` 始终直接操作原始 DataFrame 的单个元素，不会产生歧义。

### 4.4 切片包含性

- `loc` 切片包含结束标签。
- `iloc` 切片不包含结束位置。

python

```
df.loc['a':'b']   # 包含行 a 和 b
df.iloc[0:2]      # 包含位置 0 和 1，不包含 2
```

## 5. 总结：何时使用哪种方法？

| 需求                | 推荐方法         |
|:------------------|:-------------|
| 按标签选择多个行/列        | `loc`        |
| 按位置选择多个行/列        | `iloc`       |
| 快速获取或修改单个单元格的值    | `at` 或 `iat` |
| 赋值时单元格需存储列表/字典等对象 | `at` 或 `iat` |
| 按布尔条件选择行并修改某列     | `loc`        |
| 仅选择某一列            | `[]`         |
| 混合标签和位置（不推荐）      | 避免使用 `ix`    |
